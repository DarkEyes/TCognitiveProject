TCog-R: Package-Bound Mechanical Retrieval for Frame-Aware LLM Reasoning

tcog protocol v0.3 · packages/ autoload

TCog-R is not ordinary RAG. It does not ask only which chunks are relevant; it asks which frame is active, which clusters fire, which constraints govern the answer, and which trajectory should structure the response.

A working demonstration of mechanical, package-bound retrieval. The retrieval engine runs in your browser and uses no semantic similarity. Composition into prose is optionally performed by Claude, ChatGPT/OpenAI, or Gemini using your API key. Every factual claim cites back to an anchored unit; constraints surface; out-of-scope queries are refused cleanly.

Mechanically checks claim support, constraints, and frame fit using the loaded TCog packages.

Requires the selected provider key. The vanilla answer gets audited against the TCog-R trace.

Loaded packages

All .zip bundles and .json package files in the packages/ folder are loaded automatically when this page is served over http://localhost. If the page is opened as a local file, use + Load bundle and select files manually. Multiple packages can be loaded; queries route to all of them and surface frame conflicts when relevant.

Composition (optional)

Pick a provider and add its key to compose the retrieved content into prose. Without a key, the app shows raw retrieval output. Keys are stored only in browser memory and are sent only to the selected provider's API endpoint.

Scenario presets

TCog-R implements tcog-retrieval.md v0.3 mechanical mode. The retrieval engine performs cue matching, negative-cue checks, activation policy enforcement, and constraint triggering — all in JavaScript, all in your browser, all auditable. No backend. No telemetry. Packages load from the local packages/ folder.

In Mechanical only, retrieval and answer rendering are fully deterministic and auditable. In LLM-guided TCog-R, the LLM helps interpret language and select among mechanically retrieved package candidates, but TCog-R packages and the mechanical constraint gate remain authoritative: the LLM cannot invent ids, override blocking constraints, or use suppressed packages.
